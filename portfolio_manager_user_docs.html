Investment Portfolio Manager

System Authority Manual | Version 3.3

1. What This System Is (and What It Isn't)

The Investment Portfolio Manager is a headless data orchestration pipeline designed to bridge brokerage data (Schwab) with professional-grade analysis (Frontier LLMs). It provides a deterministic "spine" of truth—prices, technicals, tax lots, and written theses—while delegating complex reasoning to external models.

Explicit Non-Goals: This is not a robo-advisor (it makes no trades), not an auto-trader (it has no order execution capability), and not a backtesting engine (it analyzes real decisions made on real dates).

2. The Operating Philosophy

  • APIs calculate locally, LLMs reason externally: Python owns the math and data integrity; Claude/Gemini own the qualitative research and strategy.
  • Rotation is the unit of analysis: Most trades are treated as a pair (selling X to buy Y) rather than isolated events.
  • Small-step scaling: The system encourages incremental entries and exits (trims/adds) rather than binary all-in/all-out bets.
  • Sheets as authoritative frontend: Google Sheets is the UI. The CLI is the engine. Local JSON bundles are the immutable record.

4. The Weekly Cycle (5 minutes)

Performed every Monday morning to synchronize state:

  1. pm health: Verify tokens and API connectivity.
  2. pm snapshot --live: Freeze current holdings and enrich with technicals.
  3. pm sync transactions --live: Ingest broker trade history.
  4. pm vault sync-status: Audit thesis staleness and style drift.
  5. pm dashboard refresh --live: Update Action Zones and Valuation signals.

What to scan: Check Decision_View for red/green price highlights (Thesis triggers hit) and Tax_Control for remaining offset capacity.

5. Thesis Sync & Audit

The Thesis Sync workflow (pm vault sync) bridges the gap between structured portfolio data and your unstructured markdown theses. It allows you to keep your hand-written prose while automatically injecting the latest facts from Google Sheets.

The Managed Regions Strategy

Your vault/theses/*.md files are protected. The sync tool only mutates specific "Managed Regions" identified by HTML comment fences. Everything else—your narrative, your exit conditions, your behavioral guardrails—is never touched.

<!-- region:position_state -->
| Metric | Value |
| :--- | :--- |
| **Quantity** | 100.00 |
| **Cost Basis** | $296.97 |
...
<!-- endregion:position_state -->

Key Operations

  • Audit staleness: pm vault sync-status prints a table showing when each thesis was last reviewed and if the current weight has drifted past the style ceiling.
  • Safe synchronization: pm vault sync --ticker UNH --show-diff performs a dry-run update. It updates frontmatter (cost basis, last reviewed) and injects the latest trade log and realized P&L into the managed regions.
  • Live commit: Use --live to write changes. The tool creates a timestamped .bak file before every write for absolute safety.
  • Bootstrap regions: Use --force on a new or existing thesis to inject the managed region fences if they are missing.
Safety Guardrail: Thesis sync uses ruamel.yaml in round-trip mode. This ensures that even when the tool updates your triggers: block, your manual comments and formatting are preserved perfectly.

6. The Weekly Deep Dive

Triggered midweek when a position enters an action zone or a rotation is considered:

pm export rotation --sell TICKER --buy TICKER
  1. Navigate to exports/.
  2. Copy prompt.md to Claude 3.5 or Gemini 2.0.
  3. Attach context.json and relevant theses/ files.
  4. Record the decision outcome manually in the Decision_Log or staging enrichment.

6. The Monthly Retrospective

Conducted the first Saturday of every month to tune market timing:

pm export rotation-retrospective --last-n 20

What to look for: Patterns where specific styles (e.g. GARP) or technical setups (e.g. RSI > 70) lead to consistent underperformance or outperformance relative to the "sold" position.

Daily routine: pm morning

One command at market open:

pm morning --live

Runs health → Schwab transaction sync → snapshot → dashboard refresh, in that order. Halts immediately if Schwab tokens or Sheet access are broken (you'd want to know that before proceeding anyway). Continues past warnings unless you pass --strict. Drop --live to dry-run the whole sequence and see what it would do.

Flags

  • --live: Write to Google Sheets. Omit for a read-only dry-run preview.
  • --tx-days N: Days of transaction history to sync (default 7).
  • --strict: Halt on any health warning, not just critical failures.
  • --skip-transactions: Skip transaction sync. Use when Schwab's transaction endpoint is flaky.
  • --skip-tax: Skip the Tax_Control refresh step.
  • --skip-health: Skip the upfront health check (not recommended).
If transactions sync fails (Schwab's transaction endpoint is the flakiest part of the API), the snapshot and dashboard refresh still run. You'll see a ⚠ next to "Transactions" in the summary and you can rerun pm sync transactions --live when it's working.

8. CLI Command Reference

Install the pm shortcut once. Run pip install -e . from the repo root. After that, every command below works as pm <command>. The long form python manager.py <command> always works as a fallback if you haven't done this step.

Standard Workflow

  • health: Runs 10+ connectivity and integrity checks. No flags.
  • snapshot [--live]: Captures market state. Use --live to update dashboard.
  • sync transactions [--live] [--days N]: Pulls broker history. Default 90 days.
  • dashboard refresh [--live] [--update]: Rebuilds view tabs. --update forces a position sync.

Trade & Thesis Management

  • journal promote [--live]: Moves approved rows from Staging to permanent Trade_Log.
  • trade review [--live]: Computes historical P&L attribution for all Trade_Log entries.
  • vault sync [--live] [--ticker T] [--show-diff] [--force]: Synchronizes Sheets data into thesis markdown files.
  • vault sync-status: Audits thesis staleness and style ceiling drift.

Export Engine

  • export list: Shows all available LLM scenarios.
  • export [flags]: Generates analysis package in exports/.
  • export inspect : Previews package contents and prompt.

9. The Export Engine — LLM Reasoning Workflow

The system provides specific scenarios to ensure the LLM has exactly the context needed without drowning in noise:

  • rotation: Claude 3.5 (Best for tax/thesis nuance).
  • macro-review: Gemini 2.0 (Best for wide context integration).
  • technical-scan: Gemini 2.0 (Best for pattern recognition across 50+ rows).
  • deep-dive: Claude 3.5 / Perplexity (Best for single-stock validation).

Each package contains a prompt.md which defines Bill's investment styles and a context.json containing the deterministic "spine" data. Results should be reviewed and key takeaways recorded in the Decision_Log.

Podcast Workflow

The intended workflow is straightforward: collect raw transcripts from a small set of macro/markets podcasts, then paste a bundled file into an external LLM (Claude or ChatGPT) and ask for a synthesis. The built-in Gemini path remains available but is no longer the default — it's an opt-in flag for users who want structured allocation rows written directly to the AI_Suggested_Allocation tab.

Why transcripts first. Pasting raw transcripts into a frontier model with your own framing prompt usually produces a more useful synthesis than running them through a fixed schema. The schema path is preserved for users who want it, but the day-to-day workflow assumes you'll do the analysis yourself.

Collecting transcripts

Pull the latest episode from each configured channel:

pm podcast batch

That's a dry-run scan: it detects new episodes via YouTube RSS, downloads transcripts to data/podcast_transcripts/, and stops. No Sheet writes, no AI calls. Run it as many times as you want — the dedup log only updates when you pass --live.

To commit the dedup log so you don't reprocess the same episodes next time:

pm podcast batch --live

Other useful forms:

pm podcast list                              # show channels and recent transcripts
pm podcast batch --channel "The Compound"    # one channel only
pm podcast fetch dQw4w9WgXcQ -s "Manual"     # one specific video by ID
pm podcast clean --days 30                   # purge old transcripts

Preparing for AI analysis

Once you have a few transcripts on disk, bundle them into a single markdown file ready to paste into Claude or ChatGPT:

pm podcast bundle              # last 5 episodes
pm podcast bundle -n 10        # last 10 episodes
pm podcast bundle --since-days 14   # everything from the past two weeks

The bundle includes a pre-written analysis prompt that explains your investing style (the four codified styles, ~$550K portfolio, small-step scaling preference) and asks the receiving LLM to extract macro thesis, sector views, ticker mentions, and points of agreement/disagreement across episodes.

Strip the prompt prefix if you want to write your own:

pm podcast bundle --no-prompt

Output lands in data/podcast_bundles/. The CLI prints the absolute path and word count when it finishes — drag the file into Claude or open it in any editor and copy-paste.

Optional: built-in Gemini analysis

If you'd rather have structured allocation rows written directly to the AI_Suggested_Allocation tab in the Portfolio Sheet, use the --analyze flag:

pm podcast batch --analyze              # dry-run: print JSON, no Sheet writes
pm podcast batch --analyze --live       # full pipeline: analyze + write to Sheet

This calls Gemini with a fixed schema and writes one row per sector to AI_Suggested_Allocation, archiving the prior batch to Logs first. Target_Allocation is never touched — that tab is manual-only by design.

The GitHub Actions cron uses this path. Every Friday at 22:00 UTC, the scheduled workflow runs the equivalent of --analyze --live across all configured channels. If you're happy letting the automation handle it, you don't need to run anything manually.

Files & Data

The Portfolio Manager produces artifacts across four places: local bundles/ and data/ directories, the vault/ directory of investment docs, the Google Sheet, and GCS blobs that hold Schwab tokens. This section is the catalog — what each artifact is, what created it, what's inside, and how to handle it.

Two axes matter for every artifact:

  • Sharing risk: Use the legend below before pasting anything into an external service.
  • Persistence: Some artifacts regenerate from a fresh snapshot; others are the only copy of something Bill wrote.

Legend:

  • 🟢 Safe to share — deterministic data, no secrets, no audit role
  • 🟡 Share with care — contains identifiable position detail; fine for personal LLM use, not public posts
  • 🔴 Never share — credentials, tokens, anything in GCS blobs
  • ♻ Regenerable — can be rebuilt from upstream sources
  • 📌 Permanent — append-only audit object, manually authored, or otherwise irreplaceable

Bundles (bundles/)

Market bundle — context_bundle_*.json

Created by: pm snapshot. Bundles always write to disk; --live only affects downstream Sheet pushes.

Hash: SHA256 of canonical-JSON contents, stored in bundle_hash.

Top-level keys: timestamp_utc, bundle_hash, source, positions, calculated_technicals, calculated_fundamentals, fmp_fundamentals, cash_manual, enrichment_errors. Read core/bundle.py for the authoritative schema.

Position dict keys: ticker, description, asset_class, quantity, price, market_value, cost_basis, unrealized_gl, unrealized_gl_pct, weight_pct, daily_change_pct, is_cash, etc.

Sharing: 🟡 — your full position list, market values, cost basis. Fine for personal LLM analysis. Not for public posts.

Persistence: ♻ — regenerable. bundles/ is gitignored; old bundles can be deleted any time.

Read by: pm bundle composite, pm bundle push, pm dashboard refresh, agent commands, all tasks/build_* writers.

Verify: pm bundle verify <path>

Vault bundle — vault_bundle_*.json

Created by: pm vault snapshot. Aggregates all thesis files and research docs into a single JSON object for LLM consumption.

Contents: documents[] (prose + metadata), theses_present, theses_missing, vault_skip_log.

Sharing: 🟡 — contains your full investment reasoning database.

Persistence: ♻ — regenerable from the vault/ directory.

Composite bundle — composite_bundle_*.json

Created by: pm bundle composite. Links a specific market bundle with a specific vault bundle.

Structure: Instead of merging, it stores pointers to the component bundles and a composite_hash derived from both. It provides the get_ticker_triggers() method used by agents.

Sharing: 🟡 — the "everything" file for portfolio analysis.

Per-ticker market bundle — <TICKER>_bndl.json

Created by: tasks/enrich_atr.py and tasks/enrich_technicals.py. Caches ticker-specific enrichment data (ATR stops, RSI, Trend) to avoid redundant API calls.

Agent output — *_output_*.json

Created by: Domain agents (e.g., pm agent rebuy → rebuy_output_*.json).

Audit: Includes bundle_hash linkage to the exact market state used for the calculation, plus specific Pydantic-validated fields (e.g., van_tharp_sizing_map).

Vault (vault/)

Thesis files — vault/theses/<TICKER>_thesis.md

Created by: pm vault add-thesis --ticker X scaffolds it; you fill it in by hand. The system reads but never writes the prose.

Structure: YAML frontmatter (ticker, style, framework_preference, entry_date, last_reviewed) → prose sections (Core Thesis, Valuation & Targets, etc.) → machine-readable triggers: YAML block.

Trigger fields: fwd_pe_add_below, fwd_pe_trim_above, price_add_below, price_trim_above, discount_from_52w_high_add, revenue_growth_floor_pct, operating_margin_floor_pct, current_weight_pct.

Sharing: 🟡 — your investment reasoning. Fine for personal LLM analysis ("review my thesis on UNH").

Persistence: 📌 — your written record. Back these up. Loss = real loss.

Vault transcripts — vault/transcripts/

Purpose: Curated, permanent reference material (e.g., earnings call transcripts). Different from the transient data/podcast_transcripts/ pipeline.

Vault research — vault/research/

Purpose: Deep-dive research notes and PDF snapshots of analyst reports.

Podcast pipeline (data/)

Raw transcripts — data/podcast_transcripts/*.txt

Created by: pm podcast batch. Plain UTF-8 text ready for LLM pasting.

Sharing: 🟢 — public information.

Persistence: ♻ — purge with pm podcast clean.

Created by: pm podcast batch --analyze. 7-day rolling window.

Sharing: 🟡 — contains AI suggestions tied to your portfolio context.

Podcast bundles — data/podcast_bundles/bundle_*.md

Dedup log — data/processed_videos.json

The Logs Sheet tab provides a low-level audit trail. Higher-level audit is provided by the Bundle Hash Chain (Market Hash → Vault Hash → Composite Hash → Agent Hash), ensuring every AI decision can be traced back to the exact market state that triggered it.

Cleanup & backup

Gitignored: bundles/, data/podcast_*, exports/, .env.

Backup priorities: vault/theses/ and the manual Sheet tabs (Trade_Log, Decision_Log). Everything else is regenerable.

# Cleanup old bundles
find bundles/ -name "*.json" -mtime +30 -delete

Handing artifacts to an external LLM

Best artifacts to use, in priority order:

  1. Composite bundle (bundles/composite_bundle_*.json) — The default for "review my portfolio" prompts.
  2. Podcast bundle (data/podcast_bundles/bundle_*.md) — For "what's the macro consensus" prompts.
  3. Specific thesis (vault/theses/<TICKER>_thesis.md) — For deep-dives on one ticker.
Safety: Never paste tokens or secrets. Identifiable position detail is fine for personal use with Claude/ChatGPT, but not for public forums.

10. Tax Visibility (CPA's corner)

The Tax_Control tab is the authoritative view for planning. The **Estimated Federal Tax** is a conservative projection using rates defined in the Config tab. It does not include state taxes or NIIT.

Wash-Sale Mechanics: The system automatically flags disallowed losses from Schwab and pins them to the top of the Tax_Control table. **Tax Offset Capacity** represents the dollar amount of gains currently harvestable by realizing losses.

11. When Things Go Wrong

  • Schwab Token Expired: Run python scripts/schwab_manual_reauth.py.
  • Sheet Quota Error: The system will auto-retry. If it fails, wait 60 seconds and re-run with --live.
  • FMP 429: This is a rate limit. The system uses a 14-day local cache; wait for the quota reset or check your API key in .env.
  • Duplicate Transactions: Use sync transactions --reconcile to identify drift between Schwab and the Sheet.
  • Bundle Hash Mismatch: Occurs if files were modified after a snapshot. Run snapshot again to create a fresh immutable state.

12. Maintenance Cadence

  • Saturday: Review Thesis_Health, update thesis triggers, and run vault snapshot --live.
  • Monthly: Run trade review --live followed by export rotation-retrospective.
  • Quarterly: Review styles.json for style drift and audit Target_Allocation.
  • Annually: Move previous year's Holdings_History to an archive tab; reset Tax_Control for the new calendar year.

13. What's Deliberately Not Here

To preserve the system's "Deterministic Spine" integrity, several features are explicitly excluded:

  • Auto-trading: Human-in-the-loop is mandatory for all trade execution.
  • Auto-suggestion of rotations: The system surfaces signals (Price vs Target); Bill identifies the rotation.
  • Web UI: Rejected to keep the data footprint small and the frontend (Sheets) familiar and reliable.
