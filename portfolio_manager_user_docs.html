1. System overview

This system manages a ~$480K liquid investment portfolio (50+ positions across multiple Schwab brokerage accounts). It is a headless CLI tool — there is no web server, no automatic UI refresh, and no state held in memory between runs. Every run freezes its inputs to an immutable hashed bundle, executes deterministically, and exits.

The authoritative user interface is Google Sheets. All pipeline output lands there. Agents write only to sandbox tabs. Target allocations are manual-only. The visual layer (charts, dashboards) is a pluggable consumer of the Sheet — it never owns data or runs the pipeline.

Core principle "LLMs synthesize, APIs calculate." Python gathers all data deterministically. Agents reason over immutable snapshots. AI output is always sandboxed before any human-approved promotion. DRY_RUN defaults to true everywhere.

What the system does

  • Ingests live portfolio positions from the Schwab API (or CSV fallback)
  • Enriches positions with prices, sector, beta, yield via yFinance and FMP
  • Freezes all inputs to a tamper-evident, SHA256-hashed context bundle
  • Runs AI agents over the frozen bundle: Re-buy, Add-Candidate, New Idea Screener, Coherence Checker
  • Writes agent output to sandbox tabs in Google Sheets for human review
  • Tracks portfolio history, risk metrics, income, and transactions over time
  • Processes podcast transcripts weekly for macro allocation signals (sandboxed)

What the system never does

  • Execute trades — read-only is a structural guarantee, not a config flag
  • Write to Target_Allocation — manual-only; agents write to AI_Suggested_Allocation
  • Provide price targets or directional market predictions
  • Fetch live data inside an agent — all data is pre-gathered before the LLM is invoked

4. The bundle architecture

Every AI call is traced to an immutable input snapshot. This is the fundamental audit guarantee — not a nice-to-have. If an agent output ever looks wrong, the bundle JSON tells you exactly what the agent saw: every position, every price, every thesis file, and the exact library versions used.

Figure 2 — Bundle composition and hash chain

Why content-hash, not Drive revision ID

The audit guarantee must hold without Drive being reachable. A content hash (SHA256 of file bytes) is self-contained: you can verify a thesis file against a six-month-old bundle without making any API call. Drive revision IDs can be deleted; file bytes are permanent.

The missing thesis rule

If a position has no _thesis.md file, the vault bundler does not fail. It produces a synthetic document with thesis_present: false and logs a warning. Agents downstream see the gap and reduce their confidence accordingly. This is intentional — you can run the agents while backfilling theses.

Agent invariants

Non-negotiable — applies to every agent No price targets in any output field. No market predictions ("X will go up"). bundle_hash is a required field in every Pydantic response schema. SAFETY_PREAMBLE is auto-prepended by ask_gemini_composite() — never duplicate it in agent system prompts. All outputs are sandboxed. Target_Allocation is manual-only and unreachable from agent code.

6. Agent data channels

The agents consume a composite bundle — a single frozen JSON artifact combining two sub-bundles. The podcast pipeline is one of roughly six distinct data channels feeding the agents.

Market bundle (core/bundle.py)

Quantitative data, refreshed every snapshot:

  • Schwab API (primary): live positions, cost basis, cash balances, per-position tax treatment, L1 quotes via /marketdata/v1/quotes, OHLCV history via /pricehistory
  • yFinance: sector, beta, dividend yield, 52-week range, earnings history (supplemental enrichment)
  • FMP: forward P/E, PEG, three-statement fundamentals, earnings surprise history (via fmp_client.py)
  • Finnhub: recent news headlines per ticker
  • Manual cash entry: money market balance and yield

Vault bundle (core/vault_bundle.py)

Qualitative data, refreshed when you run vault snapshot:

  • 51 _thesis.md files (one per position): scaling state, rotation priority, bull case, exit conditions, style tag
  • Earnings transcripts: in vault/transcripts/
  • Research notes: in vault/research/
  • Investment framework files: in vault/frameworks/ — Peter Lynch GARP (lynch_garp_v1.json), Christopher Mayer 100-Bagger (100_bagger_framework.json), Carlota Perez macro cycle (Macro_super_cycle_framework.md), Van Tharp position sizing (van_tharp_position_sizing.json), Gautam Baid management quality (joys_of_compounding_framework.json), Morgan Housel behavioral principles (psychology_of_money.json). The options framework (OptionsFramework.json) lives in agents/ rather than vault/frameworks/.

Supplemental data channels

  • STAX (tasks/stax_sync.py): monthly macro allocation signal — writes to AI_Suggested_Allocation sandbox.
  • Podcast pipeline (tasks/weekly_podcast_sync.py): YouTube RSS → transcript → Gemini → AI_Suggested_Allocation sandbox. Macro context for the macro layer.
Data Channel Gap Analysis New data sources (FRED macro data, SEC EDGAR filings, etc.) become new enrichment steps that run before bundle composite. The agents read the new fields as context from the bundle; they never call the source directly.

7. Agent development & refinement

There are two distinct loops — one for refinement of existing intelligence, one for net-new agent deployment.

Refinement loop (improving existing agents)

Because every agent output is stamped with the composite_hash, you can isolate whether a change in output came from a prompt edit or from fresh data.

  1. Run the agent dry: python manager.py agent rebuy analyze --bundle latest — output lands in bundles/runs/
  2. Identify gaps: wrong signal, missing rationale, bad scaling language, ignored thesis
  3. Edit the agent's system prompt in agents/prompts/<name>_system.txt (e.g. rebuy_analyst_system.txt) or Pydantic schema in agents/schemas/
  4. Re-run against the same bundle (--bundle bundles/composite_<hash>.json) to compare apples to apples
  5. When satisfied, promote the edit and update CHANGELOG.md

Adding new agents

The pattern for adding a new agent is fully established and predictable:

1. Define what question the agent answers
2. Identify required bundle data (market vs vault)
3. Write Python pre-computation — everything deterministic lands here
4. Write Pydantic output schema in agents/schemas/<name>_schema.py
5. Write system prompt — narrative and recommendation only, NO MATH
6. Create agents/<name>.py following the rebuy_analyst.py structure
7. Register in manager.py: agent_app.add_typer(<name>_app, name="<name>")
8. Create framework file if needed: vault/frameworks/<name>_framework.json
9. Run dry, verify checklist, update CHANGELOG.md
The Math Rule If it's a number, Python calculates it. Gemini gets pre-computed facts and writes narrative, signals, and rationale.

Lessons learned — Schwab API gotchas client_from_access_functions takes NO callback_url argument. Passing one silently shifts every argument one position right and causes 'str' object is not callable at runtime. The token writer function must accept **kwargs — schwab-py passes refresh_token=... as a keyword argument on refresh. Cash appears in currentBalances.cashBalance across account objects, not in the positions array. Quote enrichment mask must guard against zero: mask = price.notna() & (price > 0).

decision gate Phase 06 — Visual Layer

Evaluate after running the CLI + Sheets workflow for 4+ weeks. Ask: "What specific question can I not answer today that I need a new tool to answer?" Options:

  • Nothing — Sheets is enough: skip Phase 06, go directly to Phase 07.
  • Charts but no interactivity: Looker Studio, direct Sheets connector, zero code, 1 day.
  • Specific interactive gap (options chain scanner, what-if rebalance): minimal FastAPI+HTMX, one page, one purpose.
  • Ad-hoc bundle exploration: Marimo analytical workbench, personal tool not a dashboard.
Do not start Phase 06 speculatively The Streamlit lesson: building a UI layer before you know what you need from it is expensive rework. Set a calendar reminder for 4 weeks after Phase 05 completes, then evaluate with a specific capability gap in hand.

11. Non-negotiable rules

Read-only guarantee The system never executes trades. The SchwabClient wrapper exposes only GET endpoints. No order or trading endpoints are imported, referenced, or reachable anywhere in the codebase. This is a structural guarantee, not a config flag.
Target_Allocation is manual-only No pipeline function, no agent, and no script ever writes to the Target_Allocation tab. Agents write to AI_Suggested_Allocation or Agent_Outputs. Promotion from sandbox to authoritative allocation is always a manual step by Bill.
DRY_RUN defaults true Every script requires an explicit --live flag to enable Sheet writes. A script that runs silently and writes to production on first execution is a bug.
Agents never browse or fetch Python gathers all data before any LLM is invoked. The LLM receives a frozen bundle and reasons over it. There is no path for an agent to make a live API call, read a web page, or access any data not in the bundle.
Single-batch gspread writes only Never write cell-by-cell. Always batch_update() or append_rows() with a full list. Check for existing fingerprints before appending. Rate limits apply.
CASH_MANUAL and QACDS are excluded from AI analysis These synthetic positions are excluded from all agent inputs, beta calculations, and allocation percentages. They appear in total portfolio value and income calculations only.

12. Troubleshooting

Bundle hash mismatch

load_bundle() raises ValueError: hash mismatch. The bundle file was modified after being written. Never edit bundle JSON files manually. Delete the corrupted bundle and rebuild: python manager.py snapshot ...

Schwab API returns zero prices

The market is closed and lastPrice from the Quotes endpoint returns 0.0. The quote enrichment mask must be price.notna() & (price > 0) — not just notna(). When all prices are zero, the Schwab Accounts API snapshot (market_value / qty) is used as fallback.

Total portfolio value ~$52K lower than Schwab's reported total

Cash is being missed. Read currentBalances.cashBalance from each securitiesAccount object and sum across all accounts. Do not rely on CASH_EQUIVALENT entries in the positions array — their structure varies by account type.

Agent output contains wrong bundle_hash

The LLM hallucinated a different hash. ask_gemini_composite() overwrites the model's bundle_hash field with the correct composite hash after parsing and logs a warning. Trust the file, not the LLM. If this warning fires frequently, check that the bundle preamble includes the explicit instruction: "You MUST include bundle_hash='...' in your response."

Schwab token expired — manual re-auth required

If the Cloud Function failed to run for more than 7 days, the refresh token is dead. Re-authorize manually:

python -c "
import schwab.auth
schwab.auth.client_from_login_flow(
    app_key='...',
    app_secret='...',
    callback_url='https://127.0.0.1',
    token_path='/tmp/token.json'
)
"
# Then upload /tmp/token.json to the GCS bucket
gsutil cp /tmp/token.json gs://portfolio-manager-tokens/token_accounts.json

Is Cash column showing True for all rows

Known bug: Is Cash can become all-True when read back from Sheets due to boolean coercion. Never filter on df['Is Cash'] == True. Always use the reliable signals:

cash_mask = (
    df['Asset Class'].astype(str).str.lower() == 'cash'
    | df['Ticker'].astype(str).str.upper().isin({'QACDS', 'CASH_MANUAL', 'CASH & CASH INVESTMENTS'})
)

FMP returning 402 / PE ratio showing 0%

Subscription limit hit. When get_historical_pe() returns an empty DataFrame, do not fall back to avg_pe = current_pe — that makes both sides of the comparison identical and produces a misleading 0% discount. Surface None explicitly and display "data unavailable" in the UI. The right signal for a broken data dependency is silence, not a silent zero.
