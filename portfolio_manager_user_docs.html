1. System Overview

The Investment Portfolio Manager is a production-grade, headless CLI tool designed to move financial data through an automated analysis pipeline. It follows a strictly linear, auditable execution model where every significant decision is backed by a cryptographically hashed "Context Bundle."

The Headless Philosophy

Unlike traditional GUI-based portfolio trackers, this system is designed for accuracy over aesthetic. It prioritizes data integrity, type-hardened calculations (Nuclear Type Enforcement), and explicit AI reasoning. The dashboard (Google Sheets) acts as a passive view layer, while the logic resides in Python and the intelligence in Gemini 2.5 Pro.

2. Architecture & Bundles

The system is built around the concept of Immutable Context Bundles. A bundle is a JSON snapshot of the world at a specific moment in time. This prevents "state leakage" where an AI agent might analyze a ticker using price data from 5 minutes ago while another agent uses data from 5 minutes in the future.

Why Bundles? By freezing data into a file with a SHA256 hash, we ensure that the AI's "rationale" is tied to a specific set of numbers. If a calculation looks wrong later, you can inspect the exact bundle the agent used.
  • Market Bundle: Positions, quantities, live prices, and cost basis. Built via manager.py snapshot.
  • Vault Bundle: Local markdown theses, YouTube transcripts, and qualitative research. Built via manager.py vault snapshot.
  • Composite Bundle: The "Master View" that joins Market and Vault data. All agents run over this file.

3. CLI Command Reference

All operations are managed through python manager.py. By default, all commands are Dry-Run (read-only).

Core Commands

snapshot

Freezes current market state from Schwab API or CSV.

python manager.py snapshot --source schwab --live
  • --source [auto|schwab|csv]: Where to pull position data.
  • --cash [float]: Manual cash balance entry.
  • --enrich-atr: Injects Average True Range (14-day) stop levels.
  • --live: Writes the snapshot to the Daily_Snapshots and Holdings_Current tabs.

analyze-all

Runs the tactical squad of AI agents over the latest composite bundle.

python manager.py analyze-all --fresh-bundle --live
  • --fresh-bundle: Regenerates market, vault, and composite bundles before running.
  • --agents [list]: Comma-separated list of specific agents to run (e.g. valuation,tax).
  • --live: Writes all agent signals to the Agent_Outputs tab in a single batch.

dashboard refresh

Maintenance command to rebuild high-level Action views.

python manager.py dashboard refresh --update --live
  • --update: Runs a live Schwab sync before rebuilding.
  • --live: Re-builds Valuation_Card, Decision_View, and applies V2 formatting.

4. Agent Tuning Guide

The system utilizes 11+ specialized AI agents. To adjust how an agent thinks—its strictness, its logic, or its style—you only need to edit its System Prompt. The Python code is just the orchestrator; the "brains" are in the text files.

How to Tune

1. Open the Prompt File path listed below.
2. Edit the plain English instructions (e.g., change "Be aggressive with TLH" to "Only suggest TLH if loss is >$5,000").
3. Save the file and re-run manager.py analyze-all. The agent will immediately adopt the new logic.

5. Maintenance Workflows

Weekly Rotation Cycle

  1. Snapshot: Capture Schwab state (python manager.py snapshot --live).
  2. Research: Update vault/theses/ with any new notes.
  3. Analyze: Run tactical squad (python manager.py analyze-all --live).
  4. Decision: Review the Decision_View tab in Google Sheets.
  5. Refine: Use python manager.py journal promote --live to move approved ideas to the log.
Pro-Tip: If the Google Sheet formatting looks "broken" (wrong colors, unreadability), run python manager.py dashboard refresh --live to restore the V2 professional UI.
