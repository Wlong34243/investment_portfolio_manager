0. Sunday Morning Cheat Sheet

This is the hardened weekly workflow for the headless pipeline.

# Step 1: Health Check — Ensure tokens and APIs are live
python manager.py health

# Step 2: Market Snapshot — Fetch Schwab, Enrich (ATR/TA/FMP/TaxLots), Push to Sheets
python manager.py snapshot --live

# Step 3: Sync Transactions — Fetch trade history and update Trade_Log_Staging
python manager.py sync transactions --live

# Step 4: Dashboard Refresh — Rebuild Valuation_Card, Decision_View, and Color-Coding
python manager.py dashboard refresh --live

# Step 5: Analyze — Ask Gemini CLI to reason about the latest dashboard state
# (e.g., "Analyze my current Decision_View for RSI extremes and MA200 proximity")
What to look at in the sheet afterwards:
  1. Decision_View tab — The primary 30-second scan point. Look for red RSI backgrounds (overbought), bold red text (below MA200), and color-coded Valuation×Trend quadrants.
  2. Valuation_Card tab — Deep dive into P/E vs trajectory for individual tickers.
  3. Tax_Control tab — Monitor YTD realized gains and wash-sale disallowed losses.
  4. Trade_Log_Staging — Review candidate rotations derived from recent transactions.

1. System Architecture

The system follows the April 2026 Architectural Pivot: "APIs calculate locally, LLMs reason externally." The local code is a deterministic data pipeline. Reasoning is performed by the operator assisted by frontier LLMs (Gemini CLI / Claude) via data exports.

Data Flow

Schwab API / CSV → Enrich (ATR/TA/FMP) → Immutable Bundle → Google Sheets → Export Engine → External LLM Reasoning

Core Invariants

  • Read-Only API
    • RuleWhy It Exists
    Deterministic LogicNo local LLM reasoning layer. All math (yields, drift %, RSI, P/E) is computed in Python before it hits the sheet or the user.
    Dry Run DefaultNothing writes to Sheets without --live. Safe to run anytime.
    Bundle ProvenanceEvery snapshot is frozen to a JSON bundle with a SHA256 hash. Exports carry this hash for auditability.
    Trading endpoints are never imported. The system is a reporting and planning aid only.

    Tech Stack

    ComponentTechnology
    OrchestrationGemini CLI (v0.39+) — Unified subagents, Plan Mode, Chapters narrative.
    PipelinePython 3.10+ via manager.py (Typer + Rich)
    Market DataSchwab API (primary), yfinance (fallback), FMP (fundamentals bake-in)
    DatabaseGoogle Sheets via gspread

    2. Core Engine (manager.py)

    manager.py is the deterministic spine of the system. Commands are grouped by domain:

    health — Pipeline Integrity Check

    python manager.py health

    Checks Schwab tokens, API connectivity, Sheet reachability, bundle age, and thesis coverage in parallel. Exit code 0=Green, 1=Critical, 2=Warning.

    snapshot — Data Ingestion & Enrichment

    python manager.py snapshot --live                              # Fetch, enrich, and push to Sheets
python manager.py snapshot --no-enrich-fmp                      # Faster snapshot, skips FMP bake-in
python manager.py snapshot --source csv --csv export.csv --live    # CSV fallback path

    Bakes FMP fundamentals, ATR stops, and technical indicators (Murphy TA) directly into the bundle before pushing to Holdings_Current.

    dashboard — Sheet Maintenance

    python manager.py dashboard refresh --live                      # Rebuild Val_Card + Decision_View
python manager.py dashboard refresh --update --live             # Snapshot + Dashboard in one go

    vault — Thesis & Research

    python manager.py vault snapshot                               # Freeze vault bundle
python manager.py vault add-thesis TICKER                      # Scaffold new thesis file
python manager.py vault thesis-audit                           # Trigger completeness report

    sync — External Feed Sync

    python manager.py sync transactions --live                      # Fetch Schwab history → Sheets

    journal — Manual Decision Tracking

    python manager.py journal promote --live                       # Approved staging → Trade_Log
python manager.py journal rotation                             # Record a sell-buy rotation event

    3. Reasoning & Analysis (Gemini CLI)

    With the decommission of local autonomous agents, reasoning is now performed via the Gemini CLI. The CLI acts as a collaborative partner, analyzing the deterministic data produced by the pipeline.

    Gemini CLI v0.39+ Workflow

    • Chapters Narrative Flow: Interactions are grouped into intent-based chapters, keeping research and execution history organized.
    • Plan Mode by Default: The agent outlines its analysis strategy before fetching data, ensuring alignment with CPA/operator guardrails.
    • Unified Subagents: Specialized investigation tasks are handled by subagents (e.g., codebase_investigator for debugging, generalist for batch data analysis).

    Common Analysis Tasks

    TaskPrompt Pattern
    Decision_View Audit "Analyze the latest Decision_View for any RSI extremes (≤30 or ≥70) and positions currently trading below their MA200."
    Thesis Alignment "Given the fundamentals baked into the latest bundle for UNH, how well does the current price align with the quantitative triggers in its thesis file?"
    Rotation Check "I'm considering selling JPIE to fund more XOM. Analyze the relative valuation and trend signals from the Valuation_Card for both."

    4. Bundle Pipeline

    Bundles are immutable JSON snapshots stored in bundles/. They serve as the "ground truth" for all reasoning and audits.

    BundleFile PatternWhat It Contains
    Market Bundle context_bundle_*.json Positions, Price, P/E, Weight, Tax Lots, ATR stops, Technicals.
    Vault Bundle vault_bundle_*.json Markdown theses and frameworks packaged for ingestion.
    Composite Bundle composite_bundle_*.json Joined Market + Vault state. The ultimate analysis input.

    5. Vault — Theses & Frameworks

    The vault lives in vault/ and provides the qualitative context for reasoning.

    Thesis Files (vault/theses/)

    One Markdown file per position (TICKER_thesis.md). Style tags (GARP, THEME, FUND, ETF) control how valuation is interpreted by the user and LLM.

    Quantitative Triggers

    The triggers: YAML block in thesis files provides hard floors/ceilings (e.g., fwd_pe_add_below). These are cited by the LLM during reasoning sessions to confirm "thesis intact" status.

    8. Troubleshooting

    Pipeline & API Errors

    ErrorCause / Fix
    401 Unauthorized (Schwab) Token expired. Run python scripts/schwab_manual_reauth.py.
    FMP 402 Payment Required Free tier limit hit or paywalled endpoint. System falls back to yfinance. Cache handles repeats.
    Bundle hash mismatch Bundle was edited manually without re-hashing. Re-run snapshot to regenerate.
    Decision_View empty signals Run python manager.py dashboard refresh --live to rebuild the view from the latest snapshot.

    9. Legacy Archive (Deprecated)

    The following systems were decommissioned in the April 2026 pivot to reduce maintenance overhead and improve reasoning quality:

    • Local AI Agent Squad: 12 specialized Python agents (Thesis, Valuation, Bagger, etc.) stashed in deprecated/agents/.
    • Agent_Outputs Tab: No longer used for live analysis. Replaced by Decision_View and Valuation_Card.
    • analyze-all command: Removed from manager.py. Replaced by decoupled snapshot and dashboard refresh.
    • Streamlit UI: Archived to archive/streamlit_legacy/. CLI + Sheets is the primary interface.
