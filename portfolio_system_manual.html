0. Sunday Morning Cheat Sheet

This is the complete weekly workflow. Copy-paste these commands in order.

# Step 1: Full analysis run — builds fresh market snapshot, runs all 7 agents, writes to sheet
python manager.py analyze-all --fresh-bundle --live

# Step 2: Refresh Decision_View with latest agent signals
python manager.py dashboard refresh --live

# Step 3: Optional — check thesis trigger completeness (backfill queue)
python manager.py vault thesis-audit
What to look at in the sheet afterwards:
  1. Agent_Outputs tab — data_quality rows first (pipeline issues), then action/alert rows (what needs attention). The HOLD SUMMARY row at the bottom condenses the "nothing happening" positions into one line.
  2. Decision_View tab — one row per position. Non-empty signal cells mean something needs attention; empty = no active signal (good).
  3. Rebuy candidates — listed in the manifest at bundles/runs/manifest_*.json. The rebuy agent writes its own JSON output, not to Agent_Outputs.

If something fails mid-run

# Re-run specific agents only (skip the slow ones that already succeeded)
python manager.py analyze-all --agents thesis,bagger --live

# Re-run without a fresh bundle (uses the bundle from the current run)
python manager.py analyze-all --agents thesis --live

1. System Architecture

The system is a headless CLI pipeline that builds immutable data snapshots, passes them to Gemini for analysis, and writes structured results to Google Sheets. No numbers are computed by the LLM — all math (yields, P/E, drift %, stop levels) is done in Python first.

Data Flow

Schwab API / CSV → Market Bundle + Vault Bundle → Composite Bundle → ATR + TA Enrichment → 3 Core Agents (+ 5 optional) → Google Sheets

Core Invariants

RuleWhy It Exists
No LLM MathYields, drift %, P/E, stop levels, position sizes are all computed in Python and passed as facts. Gemini writes narrative only.
Pydantic SchemasEvery Gemini call uses a response_schema=SomeModel. If Gemini returns malformed JSON, it fails loudly — never silently corrupt data.
Dry Run DefaultNothing writes to Sheets without --live. Safe to run anytime.
Bundle ProvenanceEvery agent output carries bundle_hash linking it to the exact market snapshot it analyzed. Manifests stored in bundles/runs/.
Van Tharp Sizing in Python1R, stop_loss_price, position_size_units are computed from ATR before any LLM call. Gemini only cites pre-computed values.

Tech Stack

ComponentTechnology
AI ModelGemini 2.5 Flash via google-genai SDK + Vertex AI ADC (re-property-manager-487122)
Market DataSchwab API (primary), yfinance (fallback), FMP free tier (fundamentals fallback)

Schema EnforcementPydantic v2 — every agent response is validated

2. Core Engine (manager.py)

manager.py is the single entry point for all operations. It groups commands into four sub-apps:

analyze-all — Weekly Analysis Orchestrator

python manager.py analyze-all --fresh-bundle --live
python manager.py analyze-all --fresh-bundle                       # dry run (no sheet write)
python manager.py analyze-all --agents valuation,macro --live      # specific agents only
python manager.py analyze-all --agents rebuy --live                # single agent

Runs agents in sequence over a single composite bundle. The default roster is 3 agents: thesis,valuation,new_idea. The retired 5 agents (rebuy, tax, concentration, macro, bagger) remain importable via --agents for ad-hoc runs. On --live, writes one batch to Agent_Outputs (archive-before-overwrite). The rebuy agent uses a legacy schema and is excluded from the batch write — its output goes to bundles/runs/ only.

snapshot — Data Ingestion

python manager.py snapshot --live                              # Schwab API → Holdings_Current
python manager.py snapshot --live --enrich-atr                 # + ATR stop levels
python manager.py snapshot --live --enrich-atr --enrich-technicals  # + Murphy TA indicators
python manager.py snapshot --source csv --file export.csv --live    # CSV fallback

vault — Thesis & Research Management

python manager.py vault snapshot                               # freeze vault bundle
python manager.py vault add-thesis TICKER                      # scaffold new thesis file
python manager.py vault thesis-audit                           # quantitative trigger completeness report

agent — Individual Agent Runs (for testing)

python manager.py agent valuation analyze --bundle latest
python manager.py agent macro analyze --bundle latest
python manager.py agent thesis analyze --bundle latest --tickers UNH,GOOG
python manager.py agent rebuy analyze --bundle latest --live   # rebuy writes its own output

dashboard — Sheet Formatting

python manager.py dashboard refresh --live
python manager.py dashboard rebuild-decision-view --live

bundle — Inspect Snapshots

python manager.py bundle composite                             # build composite from latest
python manager.py bundle inspect bundles/composite_bundle_*.json

3. Bundle Pipeline

Bundles are JSON snapshots stored in bundles/. They are immutable once created — the SHA256 hash locks the data. The pipeline builds them in order:

BundleFile PatternWhat It ContainsBuilt By
Market Bundle context_bundle_*.json All positions with price, P/E, weight, cost basis, unrealized G/L. Schwab API or CSV. core/bundle.py
Vault Bundle vault_bundle_*.json Thesis markdown files + earnings transcripts packaged as JSON documents. core/vault_bundle.py
Composite Bundle composite_bundle_*.json Joins market + vault. Contains composite_hash = SHA256(market_hash + vault_hash). All agents read this. core/composite_bundle.py

Bundle Enrichment (appended in-place, hash-safe)

TaskCommandInjects KeyUsed By
tasks/enrich_atr.py snapshot --enrich-atr calculated_technical_stops (list keyed by ticker) Macro agent (stop triggers), Rebuy agent (Van Tharp sizing)
tasks/enrich_technicals.py snapshot --enrich-technicals calculated_technicals (MA50/200, RSI-14, MACD, volume) Valuation, Rebuy, Macro, Add-Candidate agents

Both enrichment tasks run automatically during analyze-all --fresh-bundle.

4. AI Agent Reference

All agents follow the same pattern: Python pre-computes facts → Pydantic schema constrains Gemini's output → Python post-processes (overrides any math). The system prompt in agents/prompts/ is the only tuning surface.

How to change agent behavior: Edit the .txt file in agents/prompts/. No Python changes needed. Run python manager.py analyze-all --live to apply.

Investment Style Tags

Every position has a style tag set in its thesis frontmatter. Style controls how the Valuation Agent frames its signal.

Chunked Agents

These agents split positions into groups of 10 to stay under token limits. Each chunk is a separate Gemini call with INTER_CHUNK_SLEEP = 5s between calls.

  • Rebuy — uses run_chunked_analysis() (has result.candidates)
  • Macro — manual loop (collects result.positions_analyzed)
  • Bagger — manual loop (collects result.candidates_analyzed)

5. Agent_Outputs Tab Guide

The Agent_Outputs tab is written once per analyze-all --live run. The previous run's rows are archived to Agent_Outputs_Archive before the new data is written.

Column Schema (10-column compact format)

ColumnExampleNotes
run_date2026-04-20 07:52No seconds, no T/Z noise
run_id_short6ec49054First 8 chars of the UUID. Full UUID is in the manifest file.
agentvaluationWhich agent produced the row
signalaccumulateAgent's raw signal type
tickerUNHPosition or PORTFOLIO for summary rows
actionScale in 10% on each pullback…≤80 chars, never truncated mid-word
narrative2-3 sentence rationaleFull text, no length cap
scale_stepTrim 25% over 2 sessionsSizing language; empty for holds
severityactionSort order: data_quality → action → alert → watch → info
scoreemptyReserved for future conviction scores

Reading the Sheet — Row Types

SeveritySorted PositionWhat it means
data_quality Top Pipeline problem, not a portfolio finding. E.g., "97% of portfolio has unclassified sector — sector analysis skipped." Fix by running the enrichment pipeline.
action 2nd Needs attention. Accumulate signals, concentration violations, TLH candidates, tech stops triggered. These are the rows Bill opens the sheet to find.
alert 3rd High-urgency watch: ATR stop very close, macro EXIT recommendation.
watch 4th Monitor but no action yet. Macro TRIM_25PCT, thesis WATCHLIST_DOWNGRADE.
info Bottom No action. Valuation HOLD rows are collapsed into a single [HOLD SUMMARY — 43 positions] row listing all tickers in the narrative column.
The HOLD SUMMARY row: Instead of 43 identical "hold / fair value" rows cluttering the sheet, the Valuation Agent's hold rows are collapsed into one row. The narrative column lists every hold ticker: "UNH, GOOG, AMZN, XOM, ...". Individual rows are still written to the archive tab for audit.

Disagreements Tab

When analyze-all --live runs, Thesis/Valuation signal conflicts are automatically logged to the Disagreements tab for historical auditing. A disagreement is detected when the thesis screener and valuation agent issue conflicting severity signals for the same ticker (e.g., thesis says TRIM but valuation says ADD). The tab preserves one row per conflict per run.

Decision_View Integration

The Decision_View tab pulls from Agent_Outputs and shows one row per position. Signal cells are only populated when severity is action, alert, or data_quality. Empty cell = no active signal. "Not Evaluated" is not used — empty means evaluated and nothing to do.

ColumnSource
Valuation SignalLatest valuation agent row, action-severity only
Macro SignalLatest macro agent row, action-severity only
Thesis SignalLatest thesis agent row, action-severity only
Conc FlagConcentration agent — position-level flag if flagged at action severity
TLH FlagTLH if tax agent flagged this ticker as a loss-harvest candidate
Top RationaleHighest-severity narrative (valuation → macro → thesis priority)

Quantitative Triggers

The triggers: YAML block makes the Thesis Screener's verdicts machine-verifiable. When a trigger fires (e.g., price drops below price_add_below), the agent's per_position_verdict must cite it.

# See which theses need backfill
python manager.py vault thesis-audit

# Create a new thesis file from template
python manager.py vault add-thesis NVDA
Backfill priority: See vault/THESIS_BACKFILL_GUIDE.md for style-specific examples (GARP, FUND, THEME, ETF) and the honest null rule: leave a trigger null rather than inventing a value.

Utility External Connection Key Behaviors
gemini_client.py Vertex AI — Gemini 2.5 Flash ask_gemini_composite() is the canonical CLI agent call. Pass include_vault_context=False for agents that pre-compute their own context (thesis, bagger) to avoid sending 50K+ input tokens.

429 retry: waits 60s then 120s before giving up. Catches both "429" and "RESOURCE_EXHAUSTED" strings.

9. Troubleshooting

Gemini Errors

ErrorCauseFix
429 RESOURCE_EXHAUSTED Vertex AI TPM/RPM quota depleted. Most common after macro's 5 chunks exhaust the minute's budget before thesis runs. The client auto-retries at 60s and 120s. If both retries fail, re-run the specific agent: python manager.py analyze-all --agents thesis --live.
FinishReason.MAX_TOKENS + Pydantic parse failure Output token budget exceeded. Response is truncated mid-JSON. Raise the relevant GEMINI_MAX_TOKENS_* constant in config.py. Bagger and thesis are chunked — if a chunk still truncates, reduce chunk size.
Pydantic Parsing Failed: json_invalid Usually caused by MAX_TOKENS truncation. Occasionally Gemini returns non-JSON preamble. Check FinishReason in the DEBUG line just above the error. If it's MAX_TOKENS, raise the budget. If it's STOP but parsing still fails, the schema may have changed incompatibly.
Gemini returned no result ask_gemini_composite() returned None after all retries. Check the DEBUG lines above. Usually MAX_TOKENS or 429. Re-run the failed agent standalone to isolate.

Data Pipeline Errors

ErrorCauseFix
97% of portfolio has unclassified sector (data_quality row) The sector field is not populated in the market bundle positions. The Concentration agent's sector analysis is skipped when >10% unclassified. Run python tasks/enrich_fundamentals.py --live to backfill sectors. Or ensure the Schwab CSV/API positions include sector data.
FMP 402 warnings for all tickers FMP free tier doesn't support /income-statement endpoint. Normal behavior. Expected. The system falls back to yfinance. Warnings only appear on the first run per ticker — they are cached to data/fmp_cache/ and suppressed on subsequent runs.
AttributeError: 'list' object has no attribute 'get' in macro agent Older code expected calculated_technical_stops as a dict, but enrich_atr.py writes a list. Fixed in current code. Macro agent normalizes list → dict at load time. If this reappears, check that enrich_atr.py ran before the agent.
No composite bundles found Bundles directory is empty or missing. Run python manager.py snapshot --live then python manager.py vault snapshot then python manager.py bundle composite. Or use analyze-all --fresh-bundle which does all three.
Agent_Outputs tab shows old 11-column format Sheet was last written by an individual agent run (agent valuation analyze --live), not analyze-all. The compact 10-column format only applies to analyze-all batch writes. Individual agent runs still write old format. Run analyze-all --live to get the compact layout.
Schwab API returns stale positions Token expired (offline > 7 days) or Cloud Function failed to refresh. Check schwab_alert.json in GCS for alert status. Run python scripts/schwab_manual_reauth.py if token is expired.

Known Limitations

ItemStatusWorkaround
FMP earnings-surprises endpoint Returns 404 — paywalled or moved on free tier FMP_EARNINGS_AVAILABLE flag suppresses it silently after first hit. Valuation signals are based on P/E and 52-week position instead.
FMP fundamentals (income-statement) Returns 402 — paywalled on free tier yfinance provides all critical data (PE, PEG, revenue growth, gross margin, ROIC proxy). FMP cache is written on first 402 to prevent repeated calls.
Sector classification missing for most positions Ongoing — sector field not in Schwab CSV/API data Concentration agent skips sector analysis when >10% unclassified. Run enrich_fundamentals.py or populate via yfinance sector mapping.
Thesis quantitative triggers all null Template added 2026-04-20 — all existing theses predate it Run vault thesis-audit to see the backfill queue. See vault/THESIS_BACKFILL_GUIDE.md for examples by style.
enrich_fmp.py not yet built Deferred — correct fix for FMP data is to bake it into the bundle at snapshot time Currently, agents fetch FMP at run time via 7-day cache. Works, but slow on cold cache.
