Investment Portfolio Manager

Version 2.3 (Decision Review) | Operator Manual & Reasoning Guide

1. System Overview

The Investment Portfolio Manager is a production-grade, headless CLI tool designed to move financial data through an automated enrichment pipeline. Following the April 2026 architectural pivot, the system adheres to a strict mandate: APIs calculate locally, LLMs reason externally.

The Headless Philosophy

This system prioritizes accuracy and deterministic logic over aesthetic or autonomous automation. Local agents have been decommissioned in favor of a clean data pipeline that feeds frontier LLMs (like Claude, Gemini, or Perplexity) for high-quality, human-in-the-loop reasoning. The Google Sheets dashboard provides a passive, color-coded view layer for rapid 30-second scans.

2. Decision Review & Market Timing (Phase 5)

Phase 5 closes the loop on rotation-based investing. The system now automatically captures Decision Context at the moment of trade and calculates Post-hoc P&L Attribution to help you tune your market timing skill.

The Rotation Lifecycle

  1. Derive: tasks/derive_rotations.py clusters transactions into candidate rotations and captures RSI, Trend, and MA200 snapshots.
  2. Enrich: You fill in the Implicit_Bet and Thesis_Brief in the Trade_Log_Staging tab.
  3. Promote: manager.py journal promote --live moves approved rows to the permanent Trade_Log.
  4. Attribute: manager.py trade review --live calculates how those decisions performed 30, 90, and 180 trading days later.
Why Attribution? By comparing the "Buy Side" return vs. the "Sell Side" return (the Pair Return), you can see if your rotations are actually additive or if you would have been better off doing nothing.

Monthly Retrospective Workflow

Once a month, use the Export Engine to analyze patterns in your trade history:

python manager.py export rotation-retrospective --last-n 20

This packages your recent attribution data for a frontier LLM to identify timing biases (e.g., "I consistently rotate out of winners too early when RSI > 70").

How to use an Export Package

  1. Run the export command: python manager.py export rotation --sell UNH --buy VEA
  2. Navigate to the printed exports/ directory.
  3. Copy prompt.md and paste it as your first message to the LLM.
  4. Attach context.json and any files in theses/ to the chat.
  5. Review the structured reasoning and record any decisions in the Decision_Journal.

4. Tax Visibility & Control (Phase 3)

The system provides a dedicated Tax_Control view to manage realized tax liability and identify harvesting opportunities. This is not tax advice, but a planning tool built from realized trade data.

  • Net ST/LT (YTD): Realized posture for the current calendar year.
  • Est. Fed Cap Gains Tax: A conservative calculation using rates defined in the Config tab.
  • Tax Offset Capacity: The "behavioral leverage" number—exactly how much in gains can still be offset by harvesting losses.
  • Wash Sale Sentinel: Aggressive red highlighting for disallowed losses that are dragging on performance.

5. Architecture & Bundles

The system is built around Immutable Context Bundles. A bundle is a JSON snapshot of the world (prices, technicals, fundamentals, tax lots) at a specific moment in time. This ensures that any reasoning performed is tied to a frozen state, preventing "state leakage" and providing a full audit trail via SHA256 hashes.

Why Bundles? By freezing data into a hashed file, the AI's "rationale" is anchored to a specific set of facts. You can verify any analysis by inspecting the exact bundle used for that session.
  • Market Bundle: Prices, quantities, P/E, ATR stops, technical indicators, and tax lots. Built via manager.py snapshot.
  • Vault Bundle: Local markdown theses and qualitative research. Built via manager.py vault snapshot.
  • Composite Bundle: The "Master View" that joins Market and Vault data for analysis.

6. CLI Command Reference

All operations are managed through python manager.py. By default, all commands are Dry-Run (read-only) unless --live is specified.

Core Commands

health

Runs pipeline health checks (Schwab tokens, API connectivity, Sheet, bundle age, thesis coverage).

python manager.py health

snapshot

Freezes current market state, enriches with deep data (ATR/TA/FMP), and pushes to Sheets.

python manager.py snapshot --live

sync transactions

Fetches Schwab trade history and updates the Transactions tab.

python manager.py sync transactions --live

dashboard refresh

Rebuilds high-level dashboard views and applies professional formatting.

python manager.py dashboard refresh --live

Trade & Journaling

python manager.py journal promote --live

trade review

Refreshes the Rotation_Review tab with fresh P&L attribution.

python manager.py trade review --live

export rotation-retrospective

python manager.py export rotation-retrospective --last-n 20

7. Reasoning Strategy

Autonomous local agents were decommissioned in April 2026. Reasoning is now performed by the operator using Gemini CLI as a reasoning engine. This approach leverages the highest-capability frontier models while eliminating the overhead of maintaining local agent code.

Collaborative Reasoning Workflow

  1. Data Export: Use the dashboard (Google Sheets) to scan for "broken" or "interesting" positions (red RSI, price < MA200, etc.).
  2. Context Injection: Provide Gemini CLI with the data from the relevant dashboard row or thesis file.
  3. Interactive Analysis: Use Gemini CLI's Plan Mode and Chapters to walk through complex decisions (e.g., "Analyze the tax impact of rotating UNH into more VEA given our YTD gains").

8. Maintenance Workflows

Weekly Operational Cycle

  1. Check: Run python manager.py health.
  2. Ingest: Run python manager.py snapshot --live and python manager.py sync transactions --live.
  3. Derive: Run python tasks/derive_rotations.py to identify new trade clusters.
  4. Dashboard: Run python manager.py dashboard refresh --live.
  5. Analyze: Review Decision_View. Review Trade_Log_Staging and approve rotations.
  6. Record: Run python manager.py journal promote --live.

Monthly Retrospective (First Saturday)

  1. Refresh: Run python manager.py trade review --live to update attribution.
  2. Retrospective: Run python manager.py export rotation-retrospective --last-n 20.
  3. Tune: Paste into Claude/Gemini and identify behavioral biases in rotation timing.
